# Contributing

Thanks for taking the time to contribute!

## Quick start
1. Fork the repo
2. Create a feature branch: `git checkout -b feat/my-change`
3. Make changes + add tests if applicable
4. Run formatting/linting if the project has it
5. Open a Pull Request

## Guidelines
- Keep PRs focused: one feature/fix per PR
- Prefer small, reviewable commits
- If you add a dependency, explain why
- Be kind and constructive in reviews

## Reporting issues
- Search existing issues first
- Include steps to reproduce
- Include expected vs actual behavior
- Add logs/screenshots if helpful

## Security
Please do **not** open public issues for security vulnerabilities. See `SECURITY.md`.
